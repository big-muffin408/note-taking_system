## 三、后续改进建议（按优先级）

1. **【高】补 CI 流水线**：仓库无 `.github/workflows`。建议 GitHub Actions 跑 `npm run build && npm run lint && npm test` + pytest + `docker compose build`，PR 必过。现在所有单元/集成测试无需外部依赖，CI 成本很低。
2. **【高】docker-compose 中 `JWT_SECRET` 有默认回退值** `dev-jwt-secret-change-in-production`：生产部署若漏配即用已知密钥签发 token。建议去掉默认值让服务直接拒绝启动（`requireJwtSecret` 已支持），或在 compose 中标记为必填 `${JWT_SECRET:?}`。
3. **【高】迁移存量"假 app"测试**：`user.test.ts`、`document.test.ts`、`collab.test.ts`、`sync.test.ts` 在测试内自建 mock Express app，不触及生产路由（断言的是测试自己写的实现）。本次新增测试已示范真实 app + mock DB 的写法（`shares.test.ts`、`conflict.test.ts`），建议逐步迁移，特别是 document-service。
4. **【中】document-service `index.ts` 约 1300 行单文件**：建议按 notes / versions / pdf-jobs / internal 拆分路由模块，并照 user/sync 模式拆出 `app.ts` 以支持真实路由测试。
5. **【中】`adminMiddleware` 名不副实**：它只检查"已登录"，角色校验下沉到各 handler 的 `requireAdmin`。建议改名 `requireAuthenticated` 或把 DB 角色查询并入中间件，避免误用。
6. **【中】覆盖率无门槛**：Jest 已收集 coverage 但无 `coverageThreshold`。建议在 `jest.config.base.cjs` 设基线（如 60%）并逐步上调。
7. **【中】E2E 增强**：webkit 需要系统库（`sudo npx playwright install-deps webkit`，本次环境无免密 sudo 未装）；现有 e2e 全部 mock 后端，建议增加 1-2 条打真实后端的冒烟链路（注册→建笔记→编辑→分享）；UI 断言建议引入 `data-testid`，避免再因视觉重构批量失效。
8. **【低】`y-websocket/bin/utils` 经 `createRequire` 引入内部路径**：v2 起该路径不存在，锁版本或迁移到官方推荐的 `@y/websocket-server`。
9. **【低】杂项**：`scripts/start-all.sh` 在 git 索引中缺执行位（`git update-index --chmod=+x`）；无 LICENSE 文件；无 dependabot/renovate；镜像用 node:20-alpine 而本地开发是 node 22，建议统一；ai-service 的 SSE 流式与 RAG 检索路径测试薄弱。
