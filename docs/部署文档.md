# AI 协作笔记系统部署文档

## 概述

本文档描述了如何部署 AI 协作笔记系统到生产环境。

## 前置要求

### 硬件要求

- **最低配置**: 2 核 CPU, 4GB 内存
- **推荐配置**: 4 核 CPU, 8GB 内存
- **存储**: 至少 20GB 可用空间

### 软件要求

- **操作系统**: Ubuntu 20.04+ / CentOS 7+ / Debian 10+
- **Docker**: 20.10+
- **Docker Compose**: 2.0+
- **Git**: 2.0+

## 部署步骤

### 1. 克隆代码

```bash
git clone <repository-url>
cd note-taking_system
```

### 2. 配置环境变量

```bash
# 复制生产环境配置模板
cp .env.production .env

# 编辑配置文件
nano .env
```

**重要配置项**:

```bash
# 生成强密码
JWT_SECRET=$(openssl rand -base64 32)
INTERNAL_SERVICE_SECRET=$(openssl rand -hex 32)
AI_SERVICE_SECRET=$(openssl rand -hex 32)

# 数据库密码
MYSQL_ROOT_PASSWORD=$(openssl rand -base64 24)
MYSQL_PASSWORD=$(openssl rand -base64 24)
MONGO_INITDB_ROOT_PASSWORD=$(openssl rand -base64 24)

# MinIO 凭证
MINIO_ROOT_USER=$(openssl rand -hex 12)
MINIO_ROOT_PASSWORD=$(openssl rand -base64 24)

# AI 配置（可选）
AI_PROVIDER=openai                        # mock / deepseek / openai / xiaomi
AI_API_KEY=your-api-key
AI_MODEL=gpt-4                            # xiaomi 默认 mimo-v2.5-pro
AI_BASE_URL=https://api.openai.com/v1     # xiaomi 默认 https://token-plan-cn.xiaomimimo.com/v1

# 应用配置
ALLOWED_ORIGINS=https://your-domain.com
APP_BASE_URL=https://your-domain.com
SERVER_PUBLIC_URL=https://your-domain.com
```

> **注意**：`JWT_SECRET` 为必填项，docker-compose.yml 中无默认值。未设置时 `docker compose` 会直接报错退出，不会以已知的开发密钥启动。

### 3. 运行安全检查

```bash
./scripts/check-security.sh
```

确保所有检查都通过。

### 4. 构建和启动服务

```bash
# 构建所有服务
docker compose build

# 启动所有服务
docker compose up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f
```

也可以使用 `start-all.sh` 脚本一键启动：

```bash
npm run start:all                      # 基础启动
npm run start:all -- --mineru          # 启用 MinerU PDF 解析
npm run start:all -- --cuda            # 启用 CUDA GPU 支持
npm run start:all -- --no-build        # 跳过构建步骤
npm run start:all -- --logs            # 启动后输出日志
npm run start:all -- --down            # 停止所有服务
```

脚本会自动检查 Docker 环境、创建 `.env`（如不存在）、等待网关健康检查通过后输出访问地址。

**Docker Compose overlay 文件**：

- `docker-compose.yml` — 基础完整服务栈
- `docker-compose.mineru.yml` — 添加 `mineru-api` 服务（GPU/vLLM，用于 PDF 解析）
- `docker-compose.cuda.yml` — ai-service 的 CUDA GPU 支持覆盖层

### 5. 验证部署

```bash
# 检查所有服务健康状态
curl http://localhost/api/user/health
curl http://localhost/api/doc/health
curl http://localhost/api/ai/health
curl http://localhost/api/sync/health

# 协同服务健康检查需要走内部端口（Nginx 未暴露此路由）
docker compose exec -T collab-service wget -qO- http://127.0.0.1:3004/health
```

### 6. 配置反向代理 (可选)

如果需要 HTTPS，配置 Nginx 反向代理：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/certificate.crt;
    ssl_certificate_key /path/to/private.key;

    # SSL 配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    location / {
        proxy_pass http://localhost:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 支持
    location /ws/ {
        proxy_pass http://localhost:80;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 维护和监控

### 查看日志

```bash
# 查看所有服务日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f user-service
docker compose logs -f document-service
docker compose logs -f ai-service
docker compose logs -f collab-service
docker compose logs -f sync-service
```

### 备份数据

```bash
# 备份 MySQL 数据库
docker compose exec mysql mysqldump -u root -p notes > backup_$(date +%Y%m%d).sql

# 备份 MongoDB 数据库
docker compose exec mongodb mongodump --out /backup/$(date +%Y%m%d)

# 备份 MinIO 数据
docker compose exec minio mc mirror /data /backup/minio/$(date +%Y%m%d)
```

### 恢复数据

```bash
# 恢复 MySQL 数据库
docker compose exec -T mysql mysql -u root -p notes < backup_20240101.sql

# 恢复 MongoDB 数据库
docker compose exec mongodb mongorestore /backup/20240101

# 恢复 MinIO 数据
docker compose exec minio mc mirror /backup/minio/20240101 /data
```

### 更新服务

```bash
# 拉取最新代码
git pull

# 重新构建服务
docker compose build

# 重启服务
docker compose down
docker compose up -d
```

### 扩展服务

```bash
# 扩展特定服务
docker compose up -d --scale user-service=3
docker compose up -d --scale document-service=3
```

## 常见问题

### Q: 如何重置管理员密码？

A: 连接到 MySQL 数据库，更新用户密码：

```sql
UPDATE users SET password_hash = '<new-hash>' WHERE email = 'admin@example.com';
```

### Q: 如何添加新的 AI 服务提供商？

A: 编辑 `.env` 文件，配置新的 AI 服务：

```bash
AI_PROVIDER=openai
AI_API_KEY=your-api-key
AI_MODEL=gpt-4
AI_BASE_URL=https://api.openai.com/v1
```

### Q: 如何配置 SMTP 邮件服务？

A: 编辑 `.env` 文件，配置 SMTP 服务：

```bash
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
MAIL_FROM=your-email@gmail.com
```

### Q: 如何构建桌面应用安装包？

A: 使用以下命令构建 Electron 桌面应用：

```bash
npm run build:desktop              # 自动检测平台
npm run build:desktop:win          # Windows (nsis/portable)
npm run build:desktop:mac          # macOS (dmg/zip)
npm run build:desktop:linux        # Linux (AppImage/deb)
```

输出文件在 `apps/web/release/` 目录下。
